# 作業日報送信スキル

セッション完了時に、当日の作業内容を絵文字マーカー形式で整理し、Google Chat Webhook にPOSTする。Claude のチャット上でも同じ内容を表示して、記録として後から見返せるようにする。

## いつ使うか

ユーザーがセッションを終える意図を示したとき。以下のような発言が該当:

**明示的な終了**
- 「今日はここまで」「区切りにする」「一旦完了」「終了」
- 「日報送って」「レポート送って」「作業完了」

**カジュアルな終わりの挨拶**（意図が明らかなら発火してOK）
- 「ありがとう、終わります」「お疲れさま」「また明日」
- 「じゃあ今日はこれで」「閉じる」「退勤」
- 「もう寝る」「今日はもう作業しない」等

## 使わない場面（重要）

以下は**発火しない**:
- 途中の「ありがとう」「了解」「OK」だけ（作業継続中の受け答え）
- 一つの機能が完成しただけ（まだ別作業が続きそう）
- 「〜はこれでOK」等、特定の部分作業の承認

**判断基準**: 「このセッション全体を締めくくる発言か？」を見る。迷ったら先にユーザーに「日報を送ってセッションを終えますか？」と聞いて確認しても良い。

## セッション内容が薄い場合

作業がごく短時間（1-2往復）で終わった場合や、軽微な作業しかしていない場合は、日報送信せずに会話を終えてもよい。「今日は日報を送るほどの作業ではなかったので、記録はスキップします」と伝えれば十分。

## フォーマット

```
📅 YYYY-MM-DD 作業日報
🎯 テーマ1・テーマ2・テーマ3

[カテゴリ絵文字] カテゴリ名
 • 箇条書き項目1
 • 箇条書き項目2

[カテゴリ絵文字] カテゴリ名
 • 箇条書き項目1

⚠️ 残っている課題
 • 課題1
 • 課題2

🔜 次のアクション
 • アクション1
 • アクション2
```

## 手順

### 1. 日付を取得
```bash
date +%Y-%m-%d
```

### 2. テーマを抽出
当日のセッションから主要なテーマを2〜3個抽出して `・` 区切りで2行目に置く。
例: `🎯 シーン分類精度改善・ガイド作成・セキュリティ対応`

### 3. カテゴリ分類
作業内容をカテゴリ別にまとめる（各2〜4項目）。使うカテゴリ絵文字の例:
- 🔧 機能改善
- 🐛 バグ修正
- 🎨 UI/UX
- 📖 ドキュメント
- 📝 仕様・設計
- 🔒 セキュリティ
- ⚙️ 設定・インフラ
- 🧪 テスト
- 📊 データ分析
- 🚀 デプロイ・リリース

### 4. 箇条書き記法
- マーカーは ` • `（半角スペース+bullet）で詰まりを軽減
- 技術的詳細より「何を達成したか」を中心に
- 各行 30〜50文字程度に抑える
- 合計 600〜800文字以内が目安（残課題・次アクション込み）

### 4-B. 残課題セクション（⚠️）
以下から抽出して 1〜5項目で記述:
- TodoWrite の `pending` で完了しなかったタスク
- セッション中に「後日対応」「別途」「時間あるときに」と保留にしたもの
- バグ・懸念点で未解決のもの

**該当なしの場合**: このセクションを丸ごと省略して良い

### 4-C. 次のアクションセクション（🔜）
以下から抽出して 1〜5項目で記述:
- 次回セッション開始時に最初にやるべきこと
- ユーザー側で継続して実施する運用タスク（確認・監視など）
- 残課題のうち、具体的に「いつ・何を」が決まっているもの

**書き方**:
- 動詞で始める（「〜する」「〜を確認する」）
- 具体的に（「後でやる」ではなく「ログを見て異常ないか確認する」）
- 該当なしの場合はセクション省略可

### 5. Google Chat Webhook にPOST

Webhook URL は**環境変数** `CLAUDE_DAILY_REPORT_WEBHOOK` から読む。
環境変数が未設定の場合は、ユーザーにその旨を伝えて README.md のセットアップ手順を案内すること。

```bash
WEBHOOK_URL="${CLAUDE_DAILY_REPORT_WEBHOOK}"
if [ -z "$WEBHOOK_URL" ]; then
  echo "❌ 環境変数 CLAUDE_DAILY_REPORT_WEBHOOK が未設定です。README.md のセットアップ手順を参照してください。"
  exit 1
fi

curl -sS -X POST "$WEBHOOK_URL" \
  -H "Content-Type: application/json; charset=UTF-8" \
  -d '{"text":"📅 2026-04-20 作業日報\n🎯 テーマA・テーマB\n\n🔧 機能改善\n • 項目1\n • 項目2"}'
```

- 改行は JSON 文字列内で `\n`
- 日本語はそのまま（charset=UTF-8 指定）
- 成功時は `{"name":"spaces/.../messages/...","text":"..."}` が返る

### 6. Claude チャット上にも同じ内容を表示
送信した本文をそのままユーザーに見せる（記録として見返せるように）。

## 具体例

```
📅 YYYY-MM-DD 作業日報
🎯 機能改善・ドキュメント整備・リファクタリング

🔧 機能改善
 • ログイン画面の入力バリデーションを強化
 • 検索フォームのレスポンス速度を改善
 • ダークモード切替ボタンをヘッダーに追加

📖 ドキュメント
 • README にセットアップ手順を追記
 • API エンドポイント一覧の仕様書を更新

🧪 テスト
 • ユニットテストのカバレッジを 80% に引き上げ

⚠️ 残っている課題
 • 一部ブラウザでの表示崩れ調査
 • 負荷テスト未実施

 • 表示崩れの再現環境を特定する
 • 次週リリース前に負荷テストを実施する
```

## トラブル対応

- **環境変数が未設定**: ユーザーに `~/.zshrc` や `~/.bashrc` への `export CLAUDE_DAILY_REPORT_WEBHOOK="..."` 追加、`source ~/.zshrc` で反映を案内
- **送信が失敗した場合**: `curl -v` で詳細を確認。Webhook URL の `key`/`token` が有効かチェック
- **URL が漏洩した場合**: Google Chat Space の管理画面で webhook を再生成 → 環境変数を新しい値で上書き
